1. Arquitectura general del sistema

Luna Workspace es una aplicación web de gestión de proyectos tipo Kanban (similar a Trello/Kerika) que se instala como plugin de WordPress pero opera de forma completamente independiente con su propia base de datos MySQL.

Descripción de la arquitectura

  • Frontend (SPA): Una sola página HTML (index.html) de ~330KB que contiene toda la interfaz. Se comunica con el backend mediante llamadas AJAX/fetch a la API REST.
  • Backend (API REST): 13 archivos PHP en app/api/, cada uno maneja un dominio funcional (tareas, usuarios, workspaces, etc.). Todos requieren autenticación via Bearer Token.
  • Autenticación: Basada en tokens de sesión almacenados en la tabla sessions. El token viaja en el header HTTP Authorization: Bearer {token} o cookie luna_token.
  • Base de datos: MySQL admin_luna sin prefijo de tablas. Contiene 18 tablas propias de Luna.
  • WordPress: Solo actúa como instalador/activador. Una vez activa la app, WordPress no interviene en el funcionamiento diario.

3. Descripción detallada de cada archivo

Archivo principal del plugin

ArchivoTipoDescripción
luna-workspace.php WordPress Punto de entrada del plugin. Contiene el encabezado de WordPress con metadatos (nombre, versión, autor). Define constantes globales: LUNA_VERSION, LUNA_PLUGIN_DIR, LUNA_APP_DIR. Carga las clases de includes/ y registra los hooks de activación/desactivación de WordPress.

Aplicación principal — app/

ArchivoTipoDescripción y responsabilidades
index.html Frontend Interfaz completa de usuario (SPA — Single Page Application). Contiene todo el HTML, CSS y JavaScript de la aplicación en un solo archivo de ~330KB. Incluye: tablero Kanban, vista Gantt, vista de lista, panel de métricas, gestión de usuarios, workspaces, notificaciones, configuración SMTP y calendario. Se comunica con la API mediante fetch/AJAX. No requiere recarga de página para navegar.
config.php Base datos Auth Configuración central — incluido por todos los archivos de la API. Responsabilidades: (1) Manejador de errores PHP que devuelve JSON, (2) Carga credenciales desde luna-wp-config.php si existe, (3) Fallback con credenciales de producción hardcodeadas (admin_luna), (4) Función getDB() — conexión PDO singleton a MySQL, (5) Funciones de autenticación: requireAuth(), requireAdmin(), getBearerToken(), (6) Función tb($name) — agrega el prefijo a los nombres de tablas, (7) Cabeceras CORS para peticiones cross-origin.
luna-setup.php Utilidad Asistente web de configuración inicial. Permite al administrador ingresar los datos de conexión a la base de datos (host, nombre, usuario, contraseña, prefijo, URL del sitio) y los guarda en luna-wp-config.php. Incluye prueba de conexión antes de guardar. Debe eliminarse o protegerse después del primer uso.

API REST — app/api/

ArchivoTipoDescripción y endpoints
auth.php Auth API Login y logout. POST ?action=login: valida usuario/contraseña con password_verify(), crea registro en tabla sessions, devuelve token Bearer. POST ?action=logout: elimina la sesión activa. Actualiza last_login del usuario en cada acceso.
kanban.php API BD Núcleo del tablero Kanban — el archivo más extenso de la API. Gestiona: columnas (crear, editar, reordenar, eliminar), tarjetas/tareas (CRUD completo, mover entre columnas, archivar), asignación de usuarios a tareas, etiquetas, checklist, adjuntos, comentarios de chat, dependencias entre tareas. También envía notificaciones por email cuando se asigna o modifica una tarea.
workspaces.php API BD Gestión de workspaces y sus miembros. Acciones: list (listar workspaces del usuario), create (con validación de límite de licencia), update (nombre, descripción, color, ícono, imagen), delete, members (ver miembros), add_member, remove_member, switch (cambiar workspace activo), templates (plantillas de columnas).
users.php API Admin Gestión completa de usuarios. Acciones: list (todos los usuarios), me (perfil propio), create (solo admin), update (editar usuario), delete (solo admin), change_password, upload_photo, save_prefs (preferencias). Campos: nombre, email, usuario, rol (admin/member/visitor), cargo, departamento, teléfono, canal de notificación.
metrics.php API Métricas del dashboard. Devuelve: total de tareas, completadas, vencidas, por vencer esta semana, velocidad del equipo, distribución por columna, por prioridad, por miembro asignado, tareas completadas por semana (últimas 8 semanas), tareas creadas por semana. Todo filtrado por workspace.
notifications.php API Sistema de notificaciones in-app. Acciones: list (notificaciones no leídas del usuario), mark_read (marcar como leída), mark_all_read. Las notificaciones se crean automáticamente cuando se asigna una tarea, se comenta, o vence un plazo.
reminders.php Email API Recordatorios automáticos por email. Consulta tareas que vencen hoy y en los próximos 7 días, agrupa por usuario asignado, y envía un email personalizado a cada uno. Puede ejecutarse: (a) manualmente desde el panel admin, (b) automáticamente via cron job diario. Previene envíos duplicados guardando la fecha del último envío. Formato de email similar a Kerika: agrupado por workspace, con estado y fecha de cada tarea.
email.php Email Motor de notificaciones multicanal. No es un endpoint público — es una librería incluida por otros archivos. Funciones: sendSMTP() envía email via socket SMTP (sin PHPMailer), sendWhatsApp() via CallMeBot API, sendWhatsAppBusiness() via Meta API, sendTelegram() via Bot API, sendNotificationToUser() enruta al canal configurado por el usuario. Lee configuración de la tabla app_settings.
settings.php Admin API Configuración general de la app (solo admins). Acciones: guardar y leer configuración SMTP, probar envío de email (test_email), configurar WhatsApp (CallMeBot y Business), configurar Telegram Bot, probar canales de mensajería. Guarda toda la configuración en la tabla app_settings.
upload.php API Subida de archivos adjuntos. Recibe archivos via POST multipart, los guarda en app/uploads/ con nombre único, registra en la tabla attachments con referencia a la tarjeta. Valida tipo MIME y tamaño. No permite acceso a visitantes.
delete_file.php API Eliminación de archivos adjuntos. DELETE method. Verifica que el archivo pertenezca al usuario o que sea admin. Elimina el registro de la tabla attachments y el archivo físico de uploads/.
search.php API Búsqueda global en el workspace activo. GET ?q=término. Busca en títulos y descripciones de tarjetas. Mínimo 2 caracteres. Devuelve hasta 50 resultados ordenados por relevancia. Filtrado por workspace del usuario autenticado.
ping.php Utilidad Health check. Devuelve {"ok":true,"time":"..."}. Usado para verificar que el servidor PHP responde correctamente. No requiere autenticación.

4. Base de datos — Estructura y tablas

Base de datos activa: admin_luna en localhost

Usuario MySQL: Luna

Prefijo de tablas: ninguno (tablas sin prefijo: users, cards, etc.)

Charset: utf8mb4 — soporta emojis y caracteres especiales

Motor: InnoDB con soporte de claves foráneas

Tablas principales

users — Usuarios del sistema

id (PK) usernamenameemail password (hash)roleactive colorphotocargo deptnotesphone whatsapp_apikeytelegram_chat_id notification_channellast_logincreated_at

Roles posibles: admin, member, visitor. Las contraseñas se almacenan con password_hash() de PHP (bcrypt).

sessions — Sesiones activas

id (PK)user_id (FK→users) token (único)expires_atcreated_at

Cada login genera un token único. La validez es de 24 horas (configurable con SESSION_HOURS). requireAuth() verifica que el token exista y no haya expirado.

Controla qué usuarios tienen acceso a cada workspace y con qué rol. Un usuario admin del sistema tiene acceso a todos los workspaces.

5. Relación archivos ↔ Base de datos

Todos los archivos de API comienzan con require_once '../config.php' que establece la conexión PDO a admin_luna mediante la función getDB(). La conexión es un singleton (se crea una sola vez por request).

ArchivoTablas que utilizaOperaciones
config.php sessions, users SELECT para validar token en requireAuth()
auth.php users, sessions SELECT (login), INSERT (nueva sesión), DELETE (logout), UPDATE (last_login)
kanban.php cards, columns_k, card_assignees, card_checklist, card_tags, attachments, chat_messages, card_dependencies, notifications, users, workspaces CRUD completo. Todas las operaciones del tablero Kanban.
workspaces.php workspaces, workspace_members, columns_k, cards, users, workspace_templates, user_meta CRUD de workspaces. Gestión de miembros. Cambio de workspace activo.
users.php users, workspace_members CRUD de usuarios. Cambio de contraseña. Foto de perfil.
metrics.php cards, columns_k, workspaces, card_assignees, users Solo SELECT — consultas de agregación y estadísticas.
notifications.php notifications, users, cards SELECT (listar), UPDATE (marcar leída).
reminders.php cards, columns_k, workspaces, card_assignees, users, notifications, app_settings SELECT (tareas por vencer), INSERT (log notificaciones), INSERT/UPDATE (fecha último envío).
email.php app_settings SELECT y UPDATE de configuración SMTP, WhatsApp y Telegram.
settings.php app_settings SELECT y INSERT/UPDATE de todas las configuraciones de la app.
upload.php attachments INSERT al subir archivo. Guarda físicamente en app/uploads/.
delete_file.php attachments SELECT (verificar permisos), DELETE (registro y archivo físico).
search.php cards, columns_k, workspaces, workspace_members Solo SELECT — búsqueda LIKE en títulos y descripciones.

El token viaja de 3 formas:

1. Header HTTP: Authorization: Bearer {token} (método preferido)

2. Cookie: luna_token={token} (para navegadores)
